Server Setup Guide

Deploy the full newsletter pipeline on your personal server. Node.js + Express + Nodemailer + SMTP.

01 — Serve the page

Host this HTML file via Nginx or Caddy. Expose POST /subscribe to save subscribers and GET /digest/send to trigger manually.

02 — Store subscribers

Each POST writes to subscribers.json (or SQLite). Fields: name, email, topics[], focus_text, frequency, created_at.

03 — Cron job

Weekly: 0 13 * * 1   Monthly: 0 13 1 * *   Preview: 0 13 * * 0. All run at 1pm UTC (8am ET). Calls Anthropic API with web_search, generates personalised HTML, sends via SMTP.

04 — SMTP config

Set SMTP_HOST, SMTP_USER, SMTP_PASS, FROM_EMAIL in .env. Works with Postfix, Gmail SMTP, SendGrid, Mailgun relay.

// server.js — Express + Nodemailer + Anthropic SDK require('dotenv').config(); const express = require('express'); const fs = require('fs'); const mail = require('nodemailer'); const Anthropic = require('@anthropic-ai/sdk'); const app = express(); app.use(express.json(), express.static('public')); // SMTP transport const smtp = mail.createTransport({ host: process.env.SMTP_HOST, port: 587, secure: false, auth: { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS } }); // Save subscriber app.post('/subscribe', (req, res) => { const db = JSON.parse(fs.readFileSync('subscribers.json', 'utf8')); db.push({ ...req.body, id: Date.now(), created_at: new Date().toISOString() }); fs.writeFileSync('subscribers.json', JSON.stringify(db, null, 2)); res.json({ ok: true }); }); // Generate + send digest for one subscriber async function sendDigest(sub) { const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY }); const focus = sub.focus_text || sub.topics.join(', '); const period = sub.frequency === 'weekly' ? 'past 7 days' : 'past 30 days'; const msg = await client.messages.create({ model: 'claude-sonnet-4-20250514', max_tokens: 6000, tools: [{ type: 'web_search_20250305', name: 'web_search' }], messages: [{ role: 'user', content: buildPrompt(focus, period) }] }); const text = msg.content.filter(b => b.type === 'text').map(b => b.text).join('\n'); const papers = JSON.parse(text.match(/\[\s*\{[\s\S]*\}\s*\]/)[0]); const html = renderEmailHTML(sub, papers); await smtp.sendMail({ from: `"LM Digest" <${process.env.FROM_EMAIL}>`, to: sub.email, subject: `Liquid Metal Research Digest — ${new Date().toLocaleDateString('en-US',{month:'short',day:'numeric'})}`, html }); console.log(`✓ Sent to ${sub.email}`); } // Manual trigger: GET /digest/send?freq=weekly app.get('/digest/send', async (req, res) => { const freq = req.query.freq || 'weekly'; const subs = JSON.parse(fs.readFileSync('subscribers.json')) .filter(s => s.frequency === freq); for (const sub of subs) await sendDigest(sub); res.json({ sent: subs.length }); }); app.listen(3000, () => console.log('▶ LM Digest server on :3000'));

What the AI search prompt does

Instructs Claude to search arXiv, Google Scholar, ACS, Nature, Wiley, Physical Review, and Elsevier for liquid metal papers in the past 7 (or 30) days. Crucially, it searches for papers where liquid metal appears in the abstract — catching geophysics, MHD turbulence, planetary core dynamics, and rotating magnetoconvection papers. Returns a structured JSON array with real author names, specific dates, clickable DOIs, and plain-language summaries. Subscriber's primary topic is listed first.
