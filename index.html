How the digest works

LM Digest is an automated research-paper newsletter with one human review step. Each week it searches academic sources, verifies paper metadata, sends an admin preview, publishes the reviewed paper list, and emails each subscriber a personalized selection.

01 — Code structure

The private backend repository is both codebase and data store. config/domains/liquid-metal.cjs defines this digest's vocabulary, categories, ranking hints, and prompt context; Paper Express retargeted clones use generated JSON profiles in the same folder. Shared scripts handle Claude calls, DOI checks, CrossRef/arXiv metadata, and anti-hallucination gates. The Cloudflare Worker writes website submissions to private JSONL files, routes the ET schedule from one Cloudflare Cron trigger, and dispatches GitHub Actions for search and email jobs.

02 — Weekly cycle

The schedule has two phases. On Sunday morning ET, the pipeline searches CrossRef and arXiv using the baseline topic vocabulary plus anonymized reader-interest hints. Every candidate must pass relevance, metadata, and abstract checks before it reaches the admin preview. On Monday morning ET, the system reuses the reviewed preview, does no new paper search, publishes the approved website list, and emails each weekly subscriber up to five papers matched to their interests. Rolling biweekly and monthly archives are refreshed from the same reviewed weekly snapshots.

Sunday · morning ET
1Search CrossRef and arXiv with configured topic vocabulary plus anonymized reader-interest phrases
2Verify every DOI/arXiv ID, title, author list, and abstract source
3Hydrate metadata from CrossRef, arXiv, or reviewed digest history
4Store verified pool in the private repo as preview
5Email compact list plus incomplete candidates to admin
6Admin can add, edit, delete, or promote papers before Monday
Monday · morning ET
1Load edited Sunday preview from private data files
2Re-verify metadata and check arXiv preprints for publication updates
3Use the fast Claude model to interpret free-text preferences and boost matching verified papers
4Rank, select, and summarize papers for each subscriber's profile
5Send personalized email to all subscribers simultaneously
6Save the final weekly snapshot and refresh rolling biweekly/monthly archives
Scheduler

Cloudflare Cron runs the Worker once per minute. The Worker checks Eastern Time and starts jobs only at the due slots: Sunday preview at 8:00 AM, weekly Monday digest at 8:00 AM with an 8:36 AM backup, biweekly at 8:10 AM with an 8:46 AM backup, and monthly on the 1st at 8:20 AM with an 8:56 AM backup. Backup slots are second attempts; the send log prevents duplicate emails if the primary send already succeeded.

Anti-hallucination gates

Paper identity must come from CrossRef, arXiv, reviewed digest history, or an explicit admin edit. Claude fallback search is off during normal scheduled runs; when enabled manually, its candidates remain untrusted until DOI or arXiv metadata validates them. Public and email papers also need a trusted abstract source or admin review.

03 — Reader signals

Subscriber preferences shape discovery and ranking without letting the model invent papers. Free-text focus and comments become anonymized search hints in addition to the baseline search. Author-watch names trigger author-field searches. For Monday emails, topic choices and free text rank the already verified paper pool; if no preferred-topic papers are available, the subscriber still receives the top five overall papers on time. The public Digest tab always shows the full approved archive.

Ratings

Thumbs up on any paper in the Digest tab. These signals are stored privately and can guide future ranking and admin review.

Recommendations

Submit a DOI at the bottom of the Digest tab. Recommended papers are saved for admin review and boost ranking when they already appear in the weekly paper pool.

Subscriber profile

Research focus and comments help widen the next Sunday search after anonymization. Author-watch names are searched through author fields. Focus, author-watch names, affiliation, and comments also feed deterministic ranking plus guarded Claude preference scoring before sending.

04 — Technology stack

The system runs on lightweight services: GitHub Pages for the website, Cloudflare Workers for intake and scheduled dispatch, a private GitHub repo for storage, GitHub Actions for search/send jobs, SMTP for email, and CrossRef/arXiv for normal discovery. Claude is used for preference scoring and summaries, not as the source of paper metadata.

AI
Claude search model — optional fallback search, disabled during normal scheduled runs and gated by metadata checks
Claude fast model — interprets subscriber free text for preference scoring and writes summaries for the selected verified papers
CrossRef API — paper discovery, DOI verification, preprint upgrade detection, and reader-signal query expansion
arXiv API — preprint discovery, author metadata, reader-signal query expansion, and author-watch lookup
Infrastructure
Cloudflare Worker — tiny public intake API and one-cron ET scheduler without an always-on server
Private GitHub repo — append-only JSONL data store and digest snapshots
GitHub Actions — workflow jobs for Sunday preview, weekly sends, alternating biweekly sends, and monthly sends
GitHub Pages — this website, deployed on every push
SMTP — email delivery via any standard mail relay
05 — Retargeting to another field

The workflow is not tied to liquid metals. To retarget it, ask the LM Digest maintainer for access to the private Paper Express starter repo, generate one non-secret research profile, save it in a fresh private clone, and set your own GitHub, Cloudflare, SMTP, and Claude credentials outside the profile.

Retargeting builder

06 — Private data store

Subscriber data lives in private JSONL files in the backend repo. The public website never sees a GitHub token; it only talks to the Cloudflare Worker. Subscriber details are used for delivery, preference matching, anonymized search hints, recommendations, ratings, and unsubscribe/update handling.
