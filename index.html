How the digest works

This newsletter is fully automated — no manual curation. Every week an AI research librarian searches academic databases, verifies what it finds, personalises the results for each subscriber, and delivers them by email. Here is how the pieces fit together.

01 — Code structure

Five files do all the work. shared.cjs is the library — it exports the search prompt, DOI verification, CrossRef lookups, and author hydration. Both pipeline scripts require it, so there is only one copy of every critical function. server.js runs continuously on Railway and exposes the REST API that everything else calls. The two pipeline scripts run ephemerally inside GitHub Actions once a week.

File Lines Role
shared.cjs288Single source of truth — prompt, Sonnet search, Haiku calls, DOI verify, CrossRef, author hydration
server.js663Always-on Express API on Railway — subscribers, digest store, ratings, recommendations
preview.cjs215Sunday pipeline — search, verify, hydrate authors, store as preview, email admin
digest.cjs503Monday pipeline — reuse Sunday preview, merge late recommendations, rerank per subscriber, send emails
index.html~1200This page — subscribe form, digest viewer, thumbs-up ratings, DOI recommendation box
02 — Weekly cycle

The week runs in two phases separated by 24 hours. Sunday's search builds the canonical paper pool and emails a preview to the admin. Monday reuses that pool, merges any late recommendations, personalises the order for each subscriber, and sends the full digest.

Sunday · 8am ET
1Search arXiv, ACS, Nature, Wiley, Elsevier, Physical Review
2Verify every DOI resolves and title matches
3Hydrate author lists from CrossRef and arXiv
4Store verified pool in database as preview
5Email compact list to admin for review
Monday · 8am ET
1Load Sunday preview from database
2Fetch any DOIs recommended after Sunday
3Generate two-paragraph summary with Haiku
4Re-rank paper pool for each subscriber's profile
5Send personalised email to all subscribers simultaneously
03 — Self-improving search

Each week's search gets smarter based on three signals from subscribers. These are injected directly into the search prompt so the AI librarian knows what this audience values before it begins.

Ratings

Thumbs up on any paper in the Digest tab. Highly rated papers become relevance examples injected into the next week's prompt.

Recommendations

Submit a DOI at the bottom of the Digest tab. The paper is looked up, verified, and added to Monday's digest regardless of recency or journal tier.

Subscriber profile

Research focus, affiliation, and comments from your subscription form. Used by Haiku to re-rank the paper pool specifically for you before sending.

04 — Technology stack

Built entirely with open tooling and deployed on free or near-free infrastructure. The Anthropic API is the only meaningful cost — roughly one Sonnet search call per week.

AI
Claude Sonnet 4 — weekly paper search with web search tool
Claude Haiku — digest summary and per-subscriber re-ranking
CrossRef API — DOI verification and preprint upgrade detection
arXiv API — author metadata for preprints
Infrastructure
Railway — Node.js backend + PostgreSQL, always-on
GitHub Actions — cron jobs for Sunday preview and Monday digest
GitHub Pages — this website, deployed on every push
SMTP — email delivery via any standard mail relay
05 — Database

Four PostgreSQL tables on Railway. Papers are stored as JSONB so the schema never needs to change as the paper structure evolves.
